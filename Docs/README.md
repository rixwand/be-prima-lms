# Prima LMS API

All endpoints are served from the Express application under the `/api` prefix.

- **Base URL**: `/api`
- **Content type**: `application/json`
- **Authentication**: private endpoints require a `Bearer` access token generated by the login or refresh flows. Unless noted otherwise, include the header `Authorization: Bearer <access_token>`.
- **Response envelope**: successful responses wrap payloads in `{ "data": ... }`. Errors return `{ "message": string }` with an appropriate HTTP status or a validation summary in `{ "message": { field: error } }`.

The documentation is organised per module. Nested modules (e.g. `courseDomain`) appear as folders.
